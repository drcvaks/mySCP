# mySCP Private Beta Testing

This beta is for invited testers only. It is not a public launch.

## What To Test

Use the in-app Tester Checklist to track your own progress. Please also report anything confusing in Beta Feedback.

### Participant Checklist

- Log in.
- View Dashboard.
- Open Files.
- Filter Files by week, type, and scope.
- View My Chaburah.
- View chaburah discussion.
- Post a discussion message.
- View announcements.
- Try mobile navigation.
- Log out.

### Rabbi Checklist

- Open Rabbi Hub.
- Stage questions for a week.
- Stage all model questions for the selected week.
- Publish staged questions.
- Check Ask Rav questions if Ask Rav is enabled.
- Check discussion and announcements from the rabbi perspective.

### Local Admin Checklist

- Open Admin.
- Review join requests if any are pending.
- Publish a file or external link.
- Edit a published file.
- Check chaburah-only versus everyone file visibility.
- Review Beta Feedback and mark items reviewed.

### Global Admin Checklist

- Open Global Admin.
- Search for a chaburah.
- Check the current review week setting.
- Use Admin while managing a selected chaburah.
- Review Beta Feedback across roles and platforms.

## How To Report Bugs

Use the Beta Feedback page in the app. Include:

- Device: Android phone, iPhone, iPad, or Windows browser.
- Browser, if using web.
- Role: participant, rabbi, local admin, or global admin.
- Screen name: Dashboard, Files, Rabbi Hub, Admin, etc.
- What you clicked or typed.
- What you expected.
- What actually happened.

The app automatically records Web, Android, or iOS with each Beta Feedback post.

## Platform Notes

### Android

Android testers should use the EAS preview APK link shared by the developer.

Android may show an "install unknown app" warning because this is a private APK outside the Google Play Store. Testers should install it only if they trust the sender and know this is the mySCP private beta.

### iPhone and iPad

iOS testing is more controlled by Apple. TestFlight is the normal path for invited testers. Testers may need to install Apple's TestFlight app and accept an invitation.

### Windows Web

Windows testers should use the hosted web version in Chrome, Edge, or another modern browser. If the developer is running the web build locally, follow the URL provided by the developer.

## Known Limitations

- This is beta infrastructure and may be removed or hidden later.
- Email notifications are not fully wired for production delivery yet.
- Some content is still test/pilot content until final tester material is loaded.
- iOS distribution requires Apple/TestFlight setup.
- Android preview APK installs outside the Play Store and may show security warnings.
- Web hosting details depend on the chosen hosting provider.
